Java多线程学习笔记

进程和线程

  • 本节内容
    1. 进程和线程的概念
    2. 并发和并行的概念
    3. 线程的基本应用

进程和线程的概念

  • 进程

    • 程序由指令和数据组成，但这些指令要运行，数据要读写，那么就需要将指令加载至CPU，数据加载至内存。在指令运行过中还需要用到磁盘、网络等设备。进程就是加载指令、管理内存、管理IO的
    • 当一个程序被运行，从磁盘加载这个程序的代码至内存，此时就开启了一个进程
    • 进程可视为程序的一个实例。大部分程序可以同时运行多个实例进程（例如记事本、画图、浏览器等），也有的程序只有一个实例进程（例如网易云、Steam等）

  • 线程

    • 一个进程之内可以分一到多个线程
    • 一个线程就是一个指令流，将指令流中的一条条指令以一定顺序交给CPU执行
    • Java中，线程作为最小调度单位，进程作为资源分配的最小单位。在Windows中的进程是不活动 ，只是作为线程的容器

  • 二者对比

    • 进程基本上是相互独立的，而线程存在于进程内，是进程的一个子集
    • 进程拥有共享的资源，如内存空间等，供其内部线程共享
    • 进程间通信较为复杂
      • 同一台计算机的进程通信称为IPC（Inter-process Communication）
      • 不同计算机之间的进程通信，需要通过网络，并遵守共同的协议，例如HTTP
    • 线程通信相对简单，因为它们共享进程内的内存，举一个例子，多个线程可以访问同一个共享变量
    • 线程更轻量，线程上下文切换成本一般上要比进程上下文切换低

并行和并发

  • 单核CPU下，线程实际上是串行执行的。操作系统中有一个组件叫做任务调度器,将CPU的时间片分给不同的程序使用，只是由于CPU在线程间的切换非常快（因为时间片很小），所以给我们的感觉是同时运行的。总结为一句话就是：微观串行，宏观并行。

  • 小结：

    • 一般会将这种线程轮流使用CPU的做法称为并发（Concurrent）
    • 多个任务在多个处理器或者处理器核上同时执行则被称之为并行(Parallel)

多线程应用

异步调用

以方法调用角度来说，我们将需要等待结果返回，才能继续运行的，称之为同步。不需要等待结果返回，就能继续运行，称之为异步。

譬如，程序中进行一个读取磁盘的操作需要耗时5s， 如果我们没有使用多线程，在主线程中执行此操作，那么这5s中CPU什么也做不了，其他代码都得暂停执行。

因此，为了提高CPU的使用效率和程序响应速度，我们可以开辟一个新线程去处理这种比较耗时的操作，让他们在子线程中执行，从而避免阻塞主线程。

Tomcat的异步servlet也是类似的目的，让用户线程处理耗时较长的操作，避免阻塞Tomcat的工作线程

线程状态

线程五大状态

  1. 新建状态（New）
  • 触发条件：通过new Thread()创建线程对象，但未调用start()方法。
  • 特点：线程仅存在于JVM内存中，未与操作系统线程关联，不消耗CPU资源。
  1. 就绪状态

  • 触发条件：调用start（）方法后。

  • 特点：有执行资格，但没执行权，等待CPU调度。

  1. 运行状态
  • 触发条件：获得CPU时间片。
  • 特点：有执行资格，有执行权，执行run（）方法。
  1. 阻塞状态
  • 触发条件：线程执行了sleep（）或者其他阻塞式方法。
  • 特点：没有执行资格，没有执行权，阻塞方法执行完成后会变为就绪状态
  1. 死亡状态
  • 触发条件：run（）方法执行完毕，或线程因未捕获异常意外终止。
  • 特点：线程生命周期结束，不可再次启动

线程常用方法

  • setPriority(int newPriority) : 更改线程优先级
  • static void sleep(long millis) : 在指定的毫秒数内让正在执行的线程休眠
  • void join() : 等待该线程终止
  • static void yield() : 暂停当前正在执行的线程对象，并执行其他线程
  • void interrupt() : 中断线程
  • boolean isAlive() : 判断线程是否还活着

停止线程

  • 不推荐使用JDK提供的stop()、destroy()方法。因为已被废弃。
  • 建议使用一个标志位进行终止。当flag = false，则线程终止运行。
  • 注意，标志位需要volatile修饰。该关键字的作用是，当线程修改volatile值时，新值会立即刷新回主内存；其他线程读取该变量时，会强制从主内存加载最新的值。这样一来，就能够解决缓存不一致的问题。如果不用volatile修饰，变量可能被线程缓存在本地内存（如CPU寄存器），这时候其他线程就无法感知变量被修改，也就无法起到标志位的效果了。

线程休眠sleep()

  • sleep(long millis) 指定当前线程阻塞多少毫秒数
  • sleep存在异常InterruptedException
  • sleep时间达到后进入就绪状态
  • sleep可以模拟网络延时，倒计时等。
  • 每个对象都有一个锁，sleep不会释放锁

线程礼让yield()

  • 礼让线程，当前正在执行的线程调用yield()，会使该线程从运行状态转为就绪状态（不阻塞）
  • 是否礼让成功由CPU调度决定，因为不会阻塞，可能依旧是该线程抢到执行权。

线程状态

线程状态（Thread.State）,线程可以处于以下状态之一。使用getState()方法能够获取线程当前状态。

  • NEW 尚未启动的线程处于此状态
  • RUNNABLE 在Java虚拟机中执行的线程处于此状态。
  • BLOCKED 被阻塞等待监视器锁定的线程处于此状态
  • WAITING 正在等待另一个线程执行特定动作的线程处于此状态
  • TIMED_WAITING 正在等待另一个线程执行动作达到指定等待时间的线程处于此状态
  • TERMINATED 已退出的线程处于此状态。

一个线程可以在给定的时间点处于一个状态。这些状态是不反映任何操作系统线程状态的虚拟机状态。

线程优先级

java提供一个线程调度器来监控程序中启动后进入就绪状态的所有线程，线程调度器按照优先级决定应该调度哪个线程来执行。

线程优先级高仅仅代表其执行权重高，具体执行哪个还是由CPU调度决定。也就是说线程优先级高并不一定先执行，高优先级线程获得更多CPU时间片的概率更高。

线程的优先级用数字表示，范围为1~10。数字越大优先级越高。

创建的线程未主动设置优先级，默认是5。

getPriority() 用于获取线程优先级， setPriority() 用于设置线程优先级。

守护线程（daemon）

  • 线程分为用户线程和守护线程
  • 虚拟机必须确保用户线程执行完毕
  • 虚拟机不必等待守护线程执行完毕，也就是说当其他用户线程都结束了，程序会直接结束
  • 守护线程有后台记录，监控内存，垃圾回收等。
  • 通过setDaemon(true)可以将线程设为守护线程。默认线程都为用户线程，都是false。

死锁

概念 ：多个线程各自占有一些共享资源，并且都需要其他线程占有的资源才能运行，而导致两个或者多个线程都在等待对方释放资源，从而导致都停止执行的情形。某一同步块同时拥有“两个以上对象的锁”时，就可能发生“死锁”问题。

如何避免死锁

想知道如何避免死锁的发生，首先要知道死锁的必要产生条件。

产生死锁的四个必要条件：

  • 互斥条件：一个资源每次只能被一个进程使用。
  • 请求与保持条件：一个进程因请求资源而阻塞时，对已获得的资源保持不放。
  • 不剥夺条件：进程已获得的资源吗，在未使用完之前，不能强行剥夺。
  • 循环等待条件：若干进程之间形成一种头尾相接的循环等待资源关系。

Lock和synchronized的对比区别

  • Lock是显示锁（需要手动开启和关闭锁），而synchronized是隐式锁，出了作用域自动释放

  • 使用Lock锁，JVM将花费较少的时间来调度线程，性能更好。并且具有更好的扩展性（提供更多的子类）

线程协作

生产者-消费者模型：

  1. 假设仓库中只能存放一件产品，生产者将生产的产品放入仓库，消费者将仓库中产品取走消费。
  2. 如果仓库中没有产品，则生产者将产品放入仓库，如果仓库已经有产品，生产者将停止生产并等待，直到仓库中的产品被消费者取走为止。
  3. 如果仓库中放有产品吗，则消费者可以将产品取走消费，否则停止消费并等待，知道仓库中再次放入产品为止。

这是一个线程同步问题，生产者和消费者共享同一个资源，并且生产者和消费者之间互相依赖，互为条件。

这里我们就需要进行线程之间通信。

java提供了几个方法解决线程之间的通信问题

  • wait() : 表示线程一直等待，直到其他线程通知，与sleep不同，会释放锁
  • wait(long timeout) : 指定等待超时的毫秒数
  • notify() : 唤醒一个处于等待状态的线程
  • notifyAll() : 唤醒同一个对象上所有调用wait()方法的线程，优先级别高的线程优先调度。

值得注意的是：以上均为Object类的方法，都只能在同步方法或同步代码块中使用，否则会抛出lllegalMonitorStateException异常

线程池

  • 为什么需要线程池：在高并发的Java应用中，频繁创建和销毁线程是非常消耗系统资源的操作。线程池作为Java并发编程的核心组件，不仅能够复用线程、降低系统开销，还能有效控制并发数量、提升应用性能。
  • 使用线程池可以提前创建好多个线程，放入线程池，使用时直接获取，使用完放回池中。避免频繁创建销毁、实现重复利用。
  • 好处：
    • 提高响应速度（减少了创建新线程的时间）
    • 减低资源消耗（重复利用线程池中的线程，不需要每次都创建）
    • 便于线程管理：corePoolSize：核心池的大小；maximumPoolSize：最大线程数；keepAliveTime：线程没有任务最多保持多长时间会终止
  • JDK5.0起提供了线程池相关的API：ExecutorService和Executors
  • ExecutorService：真正的线程池接口。常见的子类ThreadPoolExecutor
    • void execute(Runnable command) : 执行任务/命令，没有返回值，一般用来执行Runnable
    • Future submit(Callable task):执行任务，有返回值，一般用来执行Callable
    • void shutdown() : 关闭连接池
  • Executors: 工具类、线程池的工厂类。用于创建并返回不同类型的线程池
