SpringBoot Controller层处理前端请求方法

在现代 Web 开发中，前后端分离架构已成为主流。作为连接前端用户界面和后端业务逻辑的桥梁，Controller 层的设计与编写至关重要。它负责接收、解析前端发送的 HTTP 请求，并返回处理结果。

Controller 的基本结构

一个标准的 RESTful Controller 通常由两个核心注解开始：

  • @RestController: 这是一个组合注解，相当于 @Controller + @ResponseBody。它告诉 Spring，这个类中的所有方法返回的结果都将直接被序列化为 JSON 格式并写入 HTTP 响应体中。
  • @RequestMapping("/api/users"): 这个注解定义了该 Controller 下所有 API 的公共 URL 前缀，有助于组织和管理你的 API 路由。

总结

为了方便记忆，下表总结了各种注解的适用场景：

注解 数据来源 主要用途 常用 HTTP 方法
@PathVariable URL 路径 (/users/{id}) 标识唯一资源 GET, PUT, DELETE
@RequestParam URL 查询参数 (?key=value) 过滤、分页、简单参数 GET
@RequestBody 请求体 (Request Body) 传输复杂对象 (JSON) POST, PUT
(无注解的对象) 表单数据 (Form Data) 接收表单提交 POST
@RequestHeader 请求头 (Request Headers) 获取认证令牌、元数据等 所有方法
